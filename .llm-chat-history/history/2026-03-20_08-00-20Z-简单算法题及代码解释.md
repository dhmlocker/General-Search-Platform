## 题目：Two Sum（两数之和）
给定一个整数数组 `nums` 和一个整数 `target`，找出两个不同下标 `i` 和 `j`，使得：
`nums[i] + nums[j] = target`

你可以假设：**一定存在且只有一个解**。返回这两个下标（下标从 0 开始）。

---

## 解法思路（HashMap，一次遍历）
当你枚举到下标 `i` 的元素 `x = nums[i]` 时：
- 你需要的另一个数是 `need = target - x`
- 如果之前已经见过 `need`，那就找到了答案
- 为了快速判断“之前见没见过 need”，用 `map` 记录：`值 -> 下标`

这样可以做到时间复杂度 `O(n)`。

---

## Python 代码
```python
def two_sum(nums, target):
    seen = {}  # seen[value] = index
    for i, x in enumerate(nums):
        need = target - x
        if need in seen:
            return [seen[need], i]
        seen[x] = i
```

---

## 代码解释
1. `seen = {}`：用来存每个已经遍历过的数及其下标。
2. `for i, x in enumerate(nums)`：遍历数组。
3. `need = target - x`：当前数 `x` 的“补数”，也就是需要的另一个数。
4. `if need in seen`：如果补数之前出现过，那么它的下标就是 `seen[need]`，当前下标是 `i`，直接返回 `[seen[need], i]`。
5. `seen[x] = i`：如果没找到，就把当前数记录下来，供后续元素使用。

`HashMap`（哈希表）是一种用来**快速查找/插入**数据的结构。它的核心思想是：把“键 key”通过哈希函数计算成一个“位置/桶 index”，从而在平均情况下很快定位到对应的值。

## 1) 它解决什么问题
在普通数组/列表里查找一个值，通常要从头扫一遍，时间是 `O(n)`。

HashMap 通过 `key -> value` 的映射，让你用 `key` 直接找到对应 `value`，平均时间可以做到 `O(1)`。

## 2) HashMap 的基本操作
常见操作可以理解为：

- `put(key, value)` / `seen[x] = i`：存入一对键值
- `get(key)` / `seen[need]`：用键查值
- `containsKey(key)` / `if key in seen`：判断键是否存在
- `remove(key)`：删除键值对

在 Python 里，`dict` 就是 HashMap 的一种实现；在 Java 里是 `HashMap`。

## 3) 为什么能快：哈希 + 桶
- **哈希函数**：`h = hash(key)`，把不同的 key 映射到一个桶位置
- **桶里怎么存**：如果发生“不同 key 映射到同一个位置”（哈希冲突），HashMap 会用链表/红黑树/开放寻址等策略处理

因此它不是“绝对的常数时间”，而是**平均很快**。

## 4) 时间复杂度
- 平均：`put/get/contains` 通常是 `O(1)`
- 最坏：可能退化到 `O(n)`（例如大量冲突、实现细节导致退化）
- 空间：通常需要额外内存维护桶与扩容，因此空间换时间

## 5) 用在 Two Sum 里的直观理解
在 Two Sum 里我们做的是：

- 遍历到当前数 `x = nums[i]`
- 需要的另一个数是 `need = target - x`
- 用 `seen`（HashMap）记录“之前出现过哪些数，以及它们的下标”

逻辑就是：
- 如果 `need` 已经出现在 `seen` 里，说明之前某个下标 `j` 满足 `nums[j] = need`
- 那就直接得到答案 `[seen[need], i]`

这样就避免了对每个 `x` 再去数组里找 `need`（否则会变成 `O(n^2)`）。

下面是用 `HashMap` 实现的 `Two Sum`（返回下标）的 Java 代码（平均时间复杂度 `O(n)`）：

```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> seen = new HashMap<>(); // value -> index

        for (int i = 0; i < nums.length; i++) {
            int x = nums[i];
            int need = target - x;

            if (seen.containsKey(need)) {
                return new int[] { seen.get(need), i };
            }

            seen.put(x, i);
        }

        // 题目一般保证有解；这里仅作为兜底
        return new int[0];
    }
}
```

要点很对应你之前的 `HashMap` 用法：`seen` 里保存“遍历过的数 -> 它出现的下标”，当你计算出 `need` 时，只要 `seen` 里已有它，就立刻得到答案。

这段 Java 代码用 `HashMap` 一次遍历实现 `Two Sum`，核心是“把之前出现过的数字存起来，遇到当前数字就直接查需要的另一个数”。

```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> seen = new HashMap<>(); // value -> index

        for (int i = 0; i < nums.length; i++) {
            int x = nums[i];
            int need = target - x;

            if (seen.containsKey(need)) {
                return new int[] { seen.get(need), i };
            }

            seen.put(x, i);
        }

        return new int[0];
    }
}
```

## 逐行解释
1. `Map<Integer, Integer> seen = new HashMap<>();`
   - 创建一个哈希表。
   - 它用来存两类信息：`数字值 -> 该数字出现的下标`。
   - 例如：`seen.get(7) = 之前那个 7 的下标`。

2. `for (int i = 0; i < nums.length; i++)`
   - 从左到右遍历数组。
   - `i` 是当前下标。

3. `int x = nums[i];`
   - 当前数字 `x`。

4. `int need = target - x;`
   - 如果要凑出 `target`，那么另一个数必须是：
   - `need + x = target`，所以 `need = target - x`

5. `if (seen.containsKey(need))`
   - 判断“我们之前是否已经遇到过 `need` 这个数”。

6. `return new int[] { seen.get(need), i };`
   - 如果 `need` 存在，说明：
     - `seen.get(need)` 是之前 `need` 出现的位置 `j`
     - 当前是 `i`
   - 直接返回答案 `[j, i]`。

7. `seen.put(x, i);`
   - 如果没找到 `need`，就把当前数字 `x` 记录到 `seen`，供后面元素使用。

8. `return new int[0];`
   - 兜底返回；通常题目会保证“一定有且只有一个解”，因此正常情况下不会走到这里。

## 为什么这样是 `O(n)`
- 每个元素只处理一次（一次遍历）
- `HashMap.containsKey / get / put` 平均都是常数时间 `O(1)`
- 总体平均时间复杂度就是 `O(n)`。
