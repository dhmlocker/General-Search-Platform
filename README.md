# 通用搜索平台 (General Search Platform)

## 项目概述

本项目旨在开发一个企业级聚合搜索平台，提供统一的搜索入口，帮助用户在单个页面集中搜索来自不同来源、不同类型的数据。它不仅能提升终端用户的检索效率和体验，也能通过标准化的数据接入和复用机制，显著提升企业在多个项目数据搜索功能开发时的效率和便捷性。平台采用 Vue 3 + Spring Boot 2.7 前后端分离模式开发，集成Elasticsearch实现高性能检索，并注重数据抓取、同步和架构设计，致力于打造一个高效、可扩展的现代化搜索解决方案。
## 功能介绍
### 1. 多源数据聚合搜索 (Multi-source Aggregated Search)
*   **描述：** 提供统一的搜索界面，用户可以在一个页面集中搜索来自企业内部（如文档、用户、产品）和外部（如图片、视频、第三方内容）的各类信息。支持按分类（如文章、用户、图片）进行 Tab 切换搜索，并展示聚合结果。
*   **特点：** 提升用户检索效率，避免在不同系统间切换搜索。
### 2. 智能检索与优化 (Intelligent Retrieval & Optimization)
*   **描述：** 基于Elasticsearch搜索引擎，实现对搜索内容的分词处理、关键词高亮显示、搜索建议（Suggester）以及关键词纠错等高级功能。支持模糊查询与精确查询，并提供灵活的排序机制（如按相关度、时间等）。
*   **特点：** 提高搜索结果的准确性和用户体验，实现更智能的搜索。
### 3. 标准化数据接入与扩展 (Standardized Data Integration & Scalability)
*   **描述：** 采用门面模式（Facade Pattern）、适配器模式（Adapter Pattern）和注册器模式（Registrar Pattern）设计，提供标准化的数据接口和多种数据接入方式。支持通过 HTTP Client 直接请求、JSoup 网页解析或无头浏览器抓取多种类型的数据源。
*   **特点：** 方便企业其他业务系统将数据快速接入平台，降低新搜索功能开发成本，提高系统可扩展性。
### 4. 实时数据同步与一致性 (Real-time Data Synchronization & Consistency)
*   **描述：** 实现源数据（如MySQL）与搜索索引（Elasticsearch）之间的数据实时同步，确保搜索结果的及时性和准确性。支持多种数据同步策略，包括定时任务增量同步、数据库双写机制、Logstash 数据管道以及 Canal 监听 MySQL Binlog 实时同步。
*   **特点：** 保证搜索结果与最新业务数据保持一致，提高数据鲜度。
### 5. 性能监控与稳定性保障 (Performance Monitoring & Stability Assurance)
*   **描述：** 对搜索接口进行压力测试（使用JMeter），并通过Kibana搭建可视化看板实时监控系统性能指标。在前端应用防抖（Debounce）和节流（Throttle）优化用户体验，后端采用Guava Retrying重试机制保障第三方接口调用的稳定性。
*   **特点：** 确保系统在高并发下依然稳定运行，提供流畅的用户体验。
